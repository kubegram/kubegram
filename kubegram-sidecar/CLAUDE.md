# kubegram-sidecar

eBPF-based traffic observer for Kubernetes services and ingresses.

## Overview

`kubegram-sidecar` is a Go binary with two runtime modes:

| Mode | Purpose |
|------|---------|
| `--mode=webhook` | Runs as a Kubernetes Mutating Admission Webhook server. Injects the sidecar into pods annotated with `kubegram.io/inject: "true"`. |
| `--mode=sidecar` | Runs inside injected pods. Attaches eBPF TC hooks to `eth0`, reads packet events, exposes Prometheus metrics on `:9090/metrics`, and optionally pushes stats to kubegram-server. |

## Package Layout

```
cmd/sidecar/main.go         Entry point; --mode flag routes to webhook or sidecar
pkg/types/packet.go         Shared PacketEvent type (imported by ebpf + metrics)
pkg/ebpf/
  bpf/tracer.c              eBPF C program (TC ingress/egress hooks)
  tracer_bpfel.go           Auto-generated by bpf2go — DO NOT EDIT by hand
  loader.go                 Load + attach eBPF objects; read ring buffer (Linux only)
  loader_stub.go            No-op stub for non-Linux builds
  events.go                 Re-exports types from pkg/types
pkg/metrics/prometheus.go   Prometheus collectors; HTTP parser for L7 labels
pkg/reporter/kubegram.go    Push stats to kubegram-server REST API
pkg/webhook/
  server.go                 TLS HTTPS admission webhook server
  injector.go               Pod JSON Patch builder
charts/kubegram-sidecar/    Helm chart
```

## Development Commands

```bash
make build              # Compile binary to bin/sidecar
make generate           # Recompile eBPF (requires clang + libbpf-dev on Linux)
make run-webhook        # Run webhook locally
make test               # Run unit tests
make docker-build       # Build Docker image (IMG=kubegram-sidecar:latest)
make dev-up             # docker-build + kind-load + helm-install
make dev-down           # helm-uninstall
```

## Injecting the Sidecar

Annotate any pod (or pod template in a Deployment) with:

```yaml
annotations:
  kubegram.io/inject: "true"
  kubegram.io/service: "my-service-name"   # optional; used as Prometheus label
```

## Generating eBPF Code

On a Linux host with `clang` and `libbpf-dev` installed:

```bash
make generate
```

This runs `bpf2go` which compiles `pkg/ebpf/bpf/tracer.c` and overwrites
`pkg/ebpf/tracer_bpfel.go` with the embedded BPF ELF bytecode.

## Environment Variables (sidecar mode)

| Variable | Description |
|----------|-------------|
| `KUBEGRAM_NAMESPACE` | Pod namespace (injected by webhook via fieldRef) |
| `KUBEGRAM_POD_NAME` | Pod name (injected by webhook via fieldRef) |
| `KUBEGRAM_SERVICE_NAME` | Service name label (injected from annotation) |
| `KUBEGRAM_SERVER_URL` | kubegram-server base URL (fallback for --kubegram-url) |
| `KUBEGRAM_API_TOKEN` | Bearer token for kubegram-server push |

## Environment Variables (webhook mode)

| Variable | Description |
|----------|-------------|
| `KUBEGRAM_SIDECAR_IMAGE` | Fully-qualified image to inject (default: ghcr.io/kubegram/kubegram-sidecar:latest) |
